Що таке система контролю версій (CVS)

CVS — це система, яка зберігає зміни в одному або кількох файлах так, щоб потім можна було відновити певні старі версії.

За допомогою таких систем, розробники можуть співпрацювати один з одним, не хвилюючись щодо проблем або невідповідностям в файлах.

Всі напрацювання по проекту завжди зберігаються у нас, і ми в будь-який момент можемо повернутися до старої версії або навіть скомбінувати різні рішення різних версій.

Простой пример CVS

Мы копируем файлы текущего проекта, указывая дату и время  создания копии, такой подход еще называют backup’ом, или более развитые – локальные системы контроля версий

Центральные системы контроля версий

В общем случае, делают то же что и локальные системы контроля версий, но на удаленном сервере.  Самый известный и до сих пор используемый пример ЦСУВ – Subversion(SVN)

Распределенные системы управления версиями

Примеры таких систем: git, mercurial, bazaar, darcs
Основное отличие таких систем в том, что клиенты, всегда забирают полную копию репозитория себе. Благодаря такому подходу, даже если сервер потеряет все данные, их можно моментально скопировать у кого-либо из клиентов и все снова заработает как нужно. Уничтожить данные из такой системы случайно практически невозможно.

Установка GIT

Для Windows: https://git-for-windows.github.io

Для MacOS: https://sourceforge.net/projects/git-osx-installer/

Ми рекомендуємо встановлювати розширення git bash, як окрему програму. Щоб ви не плутались з терміналом Windows.

Настройка Git

Зазвичай виконується одноразово

  1. Перевіряємо, чи працює git. Відкриваємо git bash
  2. Вводимо в консоль команду git
  3. Тепер вказуємо наше ім'я та пошту, які будуть видимі при коміті в репозиторій
  4. Перевіряємо, чи вказані наше ім'я та пошта
$ git config --global user.name "Your Name"
$ git config --global user.email your@email.com
$ git config —-list
ИЛИ
$ git config user.name

GitHub и BitBucket

Це хостинги для віддалених репозиторіїв. Або інакше – місце, де можна розташувати віддалений репозиторій.

У обох платформ є подоба соціальних мереж, можемо створювати команди та працювати над проектами з повною підтримкою pull request та issue

У кожної платформи є свої плюси або мінуси, але обидві ці платформи працюють з гітом, та нам абсолютно не важливо де ми розміщуємо код.

Ми з Вами будемо використовувати GitHub

Зареєструйтесь або увійдіть під своїм обліковим записом на сайт https://github.com Після цього, створіть новий репозиторій.
У нас з'явилась сторінка з описом як підключити наш репозиторій.

Команди, які використовуються у терміналі

  • git add . – додаємо файли в коміт. Зазвичай використовується «.» або «*» для додавання усіх файлів. Щоб додати один файл – вказуємо його ім'я та розширення.
  • git commit -m "Повідомлення, яке описує комміт" – За допомогою цієї команди ми виконуємо «коміт» змін. Тобто, вказуємо, що поточний зліпок проекту потрібно зберегти. Можна вказати без аргументу -m, в такому випадку відкриється текстовий редактор по замовчуванню, в якому потрібно буде описати коміт. Якщо це сталось випадково, вводите :q і натискаєте Enter
  • git push origin master – Пушим» або відправляємо в репозиторій наші зміпки проекту. origin – назва віддаленого репозиторію (зазвичай вибирають origin, але в теорії може бути будь-яке слово) master – назва гілки в яку додаємо зміни (master – головна гілка у репозиторії, може бути будь-якою)
  • git init ./ – ніціює пустий репозиторій в поточній директорії. Так само можна вказати будь-який шлях.
  • git remote add origin URL – Додаємо адресу віддаленого репозиторію, в якому будемо пушити зміни origin – назва віддаленого репозиторію (зазвичай вибирають origin,але в теорії може бути будь-яке слово)
  • git pull origin master – команда для отримання змін з репозиторію.
