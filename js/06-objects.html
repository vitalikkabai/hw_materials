Что такое объект?

Объект – единственный составной тип в javascript'e. Объект позволяет сохранить множество значений в виде пары: ключ -> значение. Одна из разновидностей объекта – массивы, которые мы уже рассматривали.


Объект очень удобно представлять как ящик стола, внутри которого находятся подписанные папки (ключи/keys), в которых в свою очередь лежит нужная нам информация (значения/values). (смм. подробнее)

К свойствам объекта можно получить доступ с помощью двух разных синтаксисов:


1) object["property name"] – при этом "property name" может быть абсолютно любой строкой, числом или символом


2) object.propertyName – таким образом можно обращаться к свойствам объектов, только если они соотвествуют правилам названия переменных.



Иллюстрация объекта let user = {name: "test", age: "111", likeBirds: true }



Объекты явялются ссылочным типом данных, что означает их нельзя скопировать с помощью обычной инструкции присвоения (прим. let object2 = object1).


Что же такое ссылочный тип данных? Когда мы создаем объект с помощью литерала (прим. { key: "value" }) и помещаем его в переменную происходит следующее:


Javascript резервирует в памяти некое место под наш объект и получает ссылку на эту область памяти ->

Javascript резервирует место для обычной переменной ->

Javascript запишет в переменную ссылку на место в памяти



Этот процесс необходим для того, чтобы быстро не закончилась оперативная память при выполнении какой-либо программы.


У ссылочных типов данных есть несколько особенностей:


1) При присвоении ссылочного типа данных (объект a) в другую переменную (объект b) копируется только ссылка на область в памяти. Это значит что при изменении какого-либо из объектов – изменятся оба.


2) Даже если мы очистим исходную переменную (a = undefined;), другая переменная (объект b) сохранит значение


3) Если создать два объекта с одинаковыми свойствами – они получат разные ссылки на области памяти и не будут равны. (a === c -> false)

Деструктуризация

Из объектов можно легко получить переменные с помощью специального синтаксиса деструктуризации, она позволяет одновременно создать переменные с таким же именем как свойства у объектов и присвоить переменным значения свойства. (см. пример ниже с отличиями)


Точно так же, мы можем выполнять и обратный процесс, если нам нужно создать свойство объекта с таким же именем и значением как и имя имеющейся у нас переменной.


В случае если мы хотим сохранить лишние свойства при деструктуризации, мы можем использовать оператор spread/rest let {name, age, ...other} = userObject;

Методы для обработки объектов

  • Object.keys(object) – получает все доступные ключи объекта в виде массива.
  • Object.values(object) – получает все доступные значения объекта в виде массива.
  • Object.entries(object) – получает все доступные ключи и значения объекта в виде массива из массивов в котором ключ является первым элементом массива, а значение вторым.
  • for(key in object) – цикл для перебора ключей объекта.

Методы объектов и контекст this

Функция, которая была сохранена как свойство объекта называется методом объекта. Методы отличаются от функций тем, что метод объекта всегда имеет непосредственный доступ к свойствам текущего объекта и может их изменять. (в том числе создавать новые свойства и методы, удалять их, метод даже может удалить сам себя).


Методы объектов используются в объекто-ориентированной парадигме программирования в javascript, её мы будем рассматривать подробно на следующих занятиях.
