Что такое массив

Массив – разновидность объектов в Javascript, в котором элементы хранятся в упорядоченом списке. (Например: 1 -> "хлеб", 2 -> "молоко", 3 -> "капуста")


Массивы нужны для того, чтобы можно было хранить связанные данные рядом и работать с ними одновременно. Хорошими примерами данных, которые можно сохранить в массиве будут: список продуктов, список студентов, названия городов, полученные ответы в результате опроса


Массив может хранить в себе что угодно, в том числе объекты, массивы или функции.

Массивы, которые хранят другие массивы называют многомерными.

Существует ряд задач, в которых необходимо работать с многомерными массивами.


Длинну массива можно получить с помощью свойства array.length(array – это наш массив). Это свойство показывает сколько в данный момент в массиве существует элементов.


Важно помнить, что массивы(и объекты), в отличии от простых типов данных – являются ссылочными. То есть, в переменной лишь хранится ссылка на место в памяти где лежит массив, а не сам массив.

Базовые операции над массивами

  • Создание массива можно выполнить с помощью конструкции new Array(n), где n – длинна массива или с помощью литерала массивов [1, 2, 3].

    В случае если передать в конструктор new Array(1, 2, 3) – несколько аргументов, создастся массив из переданных аргументов.

    Так же массив можно создать из любых перечисляемых объектов с помощью метода Array.from(obj).

  • Добавить элемент в конец массива можно с помощью метода .push(element). Результатом выполнения этого метода будет новая длинна массива.

  • "Вытащить" последний элемент из массива можно с помощью метода .pop(). Результатом выполнения этого метода будет последний элемент массива, при этом длинна массива уменьшится и элемент больше не будет частью исходного массива.

  • Добавить элемент в начало массива можно с помощью метода .unshift(element)

  • Убрать элемент из начала массива можно с помощью метода .shift(element)

Важно помнить, что операции с началом массива перестраивают весь массив, поэтому работают дольше и требуют больше ресурсов, чем операции в конце массива.

Другие операции над массивами

Так как массивы являются ссылочными типами данных, очень важно понимать какой метод изменяет исходный массив, а какой метод возвращает новый.


Рассмотрим несколько других полезных методов при работе с массивами:

  • Проверка вхождения в массив – .includes(element), данный метод позволяет проверить входит ли текущий элемент в массив. (см. пример ниже). Не работает с объектами!
  • Поиск индекса элемента в массиве – .indexOf(element), возвращает индекс элемента в массиве или -1 в случае если элемент не был найден.
  • Создание массива из строки – .split(divider), разбивает строку по разделителю divider и превращает в массив (см. пример ниже)
  • Превратить массив в строку – .join(divider), составляет из массива строку, добавляя разделитель divider после каждого элемента, кроме последнего.
  • Соединить несколько массивов – .concat(arr1, arr2, ..., arrN), создает новый массив из всех переданных массивов в качестве аргументов. Альтернативой этому методу может служить развертывание массивов с помощью spread оператора. [...arr1, ...arr2, ..., ...arrN]
  • Отрезать часть массива – .slice(startIndex, endIndex) – позволяет обрезать массив в нужных нам значения от startIndex до endIndex (включительно).
  • Заменить или вырезать часть массива – .splice(startIndex, cutLength, ...newElements?) – вырезает элементы начиная с индекса startIndex до startIndex + cutLength, все последующие параметры – будут вставлены на место вырезанных элементов. Этот метод изменяет исходный массив, будьте осторожнее
